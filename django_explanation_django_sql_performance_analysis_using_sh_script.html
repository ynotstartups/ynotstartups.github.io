Tiger’s two cents

I think the fundamental cause is the round-trip time excluding the process time in remote-db i.e. the time needed for a network package goes from our laptop to remote-db and back.

before you start, remove all data in the database

connect to postgres sqll

  • for local db, make psql
  • for remote db, use matt’s script in oneview-infra, python scripts/connect_db.py --developer your_name

drop and recreate database oneview

 \c postgres;  # by default, postgres connects to database called `oneview` and next drop command will fail
drop database oneview with (force);
create database oneview;

Then restart django container, you should see migrations are being applied.

how to log all django sql

  1. enable database logs which logs sql duration and sql query
"django.db.backends": {
    "handlers": ["console"],
    "level": "DEBUG",
    "propagate": False,
},
docker exec --env -t -i oneview-django-1 poetry run python manage.py import_curo_data "oneview-admin@saltus.co.uk" --use_test_accounts 2> make_data_quick_run_after_jeremy_changes.txt > make_data_quick_run_after_jeremy_changes.txt
